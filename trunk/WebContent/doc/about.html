zTree 介绍

概述

    最近对 JQuery 进行了入门，一时兴起写了一个Tree插件，供大家学习和使用，毕竟是本人第一个公开的组件，肯定有许多问题和不足之处，请大家把发现的问题，或好的想法及时与我沟通，在这里特别要感谢 独上太行 的大力支持，架构方面给了我很多关键性的建议。

    同时欢迎利用此版制作其他发行版以方便广大 zTree 爱好者，转载请保留版权信息，谢谢。

下面介绍一下zTree 的主要功能：

 1、兼容 IE、FireFox 浏览器（对于IE 6.0的兼容问题将不做特殊维护）

 2、在一个页面内可同时生成多个 Tree 实例

 3、支持 JSON 数据

 4、支持一次性静态生成 和 Ajax 异步加载 两种方式

 5、支持多种事件响应及反馈

 6、支持 Tree 的节点移动、编辑、删除

 7、支持极其灵活的 Checkbox & Radio 选择功能

 8、支持任意更换皮肤（依靠 css ）

 9、个性化图标（依靠 css ）

 10、简单的参数配置 实现 灵活多变的功能

zTree v2.0 更新记录

概述

 针对 v1.x 版本的重大修改请参考 速查表 有 # 标记的内容。

 新功能对应各项属性、方法请参考 速查表 有 * 标记的内容。

 1、更名 getCurNode() 方法为 getSelectedNode() 方法。

 2、更名 getSelectedNodes() 方法为 getCheckedNodes() 方法。

 3、增加 getNodeIndex(treeNode) 方法，获取某节点在同一层级节点中的序号（从0开始）。

 4、checkbox、radio的 onCheck 事件更名为 onChange 事件。

 5、radio只能选择单个节点，并且自动选中最新节点，完全满足radio标准，删除最大数量限制功能。

 6、修正对于节点 name 属性的显示方法，不用再担心<>等特殊字符影响效果，尤其是 js 注入。

 7、在 editable=true 状态时，能够点击编辑、删除按钮，实现名称编辑 和 快速删除节点。

 8、针对Click、Check、Drag、Drop、Rename、Del等事件都提供 before方法，让用户根据自己的需求确定是否允许该事件。

 9、修改Callback 回调函数结构，允许用户通过配置自定义各种回调函数。
